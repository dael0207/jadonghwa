# R008 — Program Generation v1

## 1. 목적

Program Generation Engine은 선택된 자동화·지원 후보를 개발팀이 검토하고 구현할 수 있는 **추적 가능한 솔루션 패키지**로 변환한다.

AI가 임의로 기능을 상상하는 것이 아니라, 모든 핵심 요구사항을 확인된 업무 모델과 사용자 결정에 연결한다.

---

## 2. 생성 성숙도

- `G0 Recommendation`: 해결 방향과 검증 실험
- `G1 Solution Blueprint`: PRD·UX·데이터·API·수용 기준
- `G2 Starter Scaffold`: 저장소 구조와 비즈니스 로직 인터페이스
- `G3 Sandbox Prototype`: 샘플 데이터 기반 제한 프로토타입
- `G4 Production Release`: 운영 배포와 모니터링

R010 MVP는 **G1까지**를 필수 범위로 한다. G2는 템플릿형 코드 뼈대를 선택 기능으로 남기며, 실제 시스템 자격증명이나 운영 배포는 하지 않는다.

---

## 3. 입력

- 확정된 `AS_IS` 업무 모델
- 선택된 Automation Candidate
- 사용자가 선택한 해결 방식
- 자율성·승인 경계
- 보안·개인정보·권한 제약
- 효과 목표
- 기존 기술 환경(알려진 경우)
- 미확정 가정 목록

## 5. 추적성 규칙

각 요구사항은 다음 메타데이터를 가진다.

```json
{
  "requirement_id": "FR-012",
  "statement": "사용자는 전사된 텍스트를 수정한 뒤 확정해야 한다.",
  "source_refs": ["claim-38", "consent-policy-2"],
  "status": "CONFIRMED",
  "risk_refs": ["risk-privacy-1"],
  "acceptance_refs": ["AC-019"]
}
```

근거가 없는 생성 기능은 `PROPOSED`로 표시하며 확정 요구사항과 분리한다.

## 6. 솔루션 템플릿

- 규칙 기반 워크플로
- RPA 자동화
- 문서 추출·검토
- 지식 검색·AI 보조
- 승인형 에이전트
- 내부 CRUD/케이스 관리 도구
- 시스템 통합 서비스
- 보고서·대시보드

템플릿은 고정 답안이 아니라 누락 방지 체크리스트로 사용한다.

---

## 7. 생성 전 게이트

다음이 없으면 `G1` 완료로 표시하지 않는다.

- 선택된 문제와 사용자
- 업무 경계
- 핵심 입력·출력
- 사람·시스템 권한
- 정상·예외 흐름
- 개인정보·보안 분류
- 수용 기준
- non-goals
- 미확정 가정

---

## 8. 금지 사항

- 확인되지 않은 API나 제품 기능을 존재한다고 가정
- 실제 자격증명을 산출물에 포함
- 법률·의료·재무 판단을 운영 규칙으로 임의 확정
- ROI를 보장
- 사용자의 승인 없이 자율성 수준 상향
- 근거 없이 “AI 에이전트가 최적”이라고 결론
- 테스트·감사·복구 없이 자동 실행 설계
